# Security Policy

## Supported Versions

Security fixes are provided for the latest commit on `main`.

## Reporting a Vulnerability

Please report vulnerabilities privately by contacting the maintainers directly.

Include:

- A clear description of the issue
- Reproduction steps or proof of concept
- Affected versions/commit hash
- Potential impact

Please do not open a public GitHub issue for suspected vulnerabilities.

## Response Expectations

- Initial acknowledgement target: within 5 business days
- Status updates: as fixes are prepared and validated
- Disclosure: coordinated disclosure after a fix is available
